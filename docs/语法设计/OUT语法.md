> 架构版本: v0.1 (设计阶段)
> 适用范围: OUT 语法域
> 配套文档: 详见 01-总纲与哲学.md、通用语法.md、INPUT语法.md

## 1. 语法域概述

OUT 区与 INPUT 区互为镜像——INPUT 把数据从外部搬进变量，OUT 把数据从变量送到外部。

OUT 区执行完毕后，整个文件执行流程结束。

引用方向：OUT 可引用 TOOLS 区函数（装饰器）和所有区域中用 `GOOUT` 关键词声明的产出变量。

### 1.1 数据来源

OUT 可以引用的变量分为两类：

| 来源 | 生命周期 | 说明 |
|------|----------|------|
| **常量**（任意区域产出） | 文件级持久 | INPUT 常量、或任何区域声明的 `CONST` 变量 |
| **`GOOUT` 标注变量**（任意区域产出） | **交付即终结** | 被 `GOOUT` 关键词标注的变量。任何区域（除 INPUT）均可使用 `GOOUT` 标注。在 OUT 成功写出后立即释放，不是常量 |

常量可以在 OUT 区反复引用多次。`GOOUT` 标注的变量则不同——它处于"持久引用"状态，直到 OUT 将其成功发出，确认信息完整输出的那一瞬间，该变量立即终结。

```
TASK :
    CALL : compute() => GOOUT result : i32
    # result 不是常量——它等 OUT 发出去后自动释放

OUT :
    result => HTTP : "https://api.com/upload"
    # result 成功发出→终结
```

## 2. 区声明

```
<OUT区>     : OUT : { <交付声明>+ }
```

---

## 3. 交付声明模板

```
<交付声明>    : <数据源> <赋值箭头> <目标关键词> : <地址>
                    [ ( <参数列表> ) ]
                    [ <装饰器> ]
                    [ { <交付声明>+ } ]

<目标关键词> : HTTP | TCP | FILES | ...
<赋值箭头>  : = | => | <=
```

### 3.1 与 INPUT 的对称关系

```
INPUT :  HTTP : "url" => RAW : bytes      # 源 → 变量
OUT   :  RAW => HTTP : "url"              # 变量 → 目标
```

INPUT 是**数据流入**，关键词在前、赋值箭头指向变量。
OUT 是**数据流出**，变量在前、赋值箭头指向关键词。

### 3.2 各部分含义

| 部分 | 必填 | 说明 |
|------|------|------|
| `<数据源>` | **是** | 待交付的变量名（引用 TASK 的 `OUT :` 产出） |
| `<赋值箭头>` | **是** | `=` COPY 或 `=>`/`<=` MOVE 到目标 |
| `<目标关键词>` | **是** | 数据送达的目标类型（HTTP、FILES、TCP……） |
| `<地址>` | **是** | 目标地址（URL、文件路径等） |
| `(参数列表)` | 否 | 目标专属参数 |
| `[装饰器]` | 否 | 在发出前加工数据 |
| `{交付声明}` | 否 | 嵌套子交付，纯语义组织 |

### 3.3 装饰器在 OUT 中的数据流

与 INPUT 对称但方向相反：

```
数据源变量  →  装饰器加工  →  目标协议 (HTTP/FILES/TCP...)
    │              │                │
    │  [encrypt] [gzip] ...         │
    │              │                │
    └── RAW ───────┘                └── HTTP : "url"
```

```
OUT :
    RAW [encrypt] [gzip] => HTTP : "https://api.com/upload"
    # ① RAW 中的数据
    # ② encrypt 加密
    # ③ gzip 压缩
    # ④ 发到目标 URL
```

## 4. 目标关键词

OUT 的目标关键词与 INPUT 的源关键词**共用同一套词表**，语义翻转：

| 关键词 | INPUT 语义 | OUT 语义 |
|--------|-----------|----------|
| `HTTP` | 从 HTTP 读 | 发送到 HTTP |
| `TCP` | 从 TCP 读 | 发送到 TCP |
| `FILES` | 从文件读 | 写入文件 |
| `DB` | 从数据库查 | 写入数据库 |
| `OSS` | 从对象存储读 | 上传到对象存储 |
| ... | ... | ... |

地址格式与参数完全复用 INPUT 附录中的定义，不再重复。

---

## 5. 示例

### 5.1 基础

```
OUT :
    RESULT => HTTP : "https://api.com/upload"
```

### 5.2 带装饰器

```
OUT :
    RAW [encrypt] [gzip] => HTTP : "https://api.com/upload"
        (method=POST, timeout=30s)
```

### 5.3 写文件

```
OUT :
    REPORT => FILES : "/tmp/output.csv"
```

---

## 6. 非法情况

| 情况 | 说明 |
|------|------|
| 非目标关键词出现在 `OUT : {}` 内 | 在 OUT 中写 `CALL` 或 `TASK` |
| 引用未用 `GOOUT` 声明的变量 | `OUT : { x => HTTP : "url" }`（x 未用 GOOUT 标注） |
| 变量名不是 `GOOUT` 标注的产出变量 | 只能交付已被 `GOOUT` 声明的变量 |
