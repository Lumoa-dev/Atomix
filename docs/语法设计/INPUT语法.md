# Atomix INPUT 语法

> 架构版本: v0.1 (设计阶段)
> 适用范围: INPUT 语法域
> 配套文档: 详见 01-总纲与哲学.md、通用语法.md

---

## 1. 语法域概述

INPUT 区声明数据的外部来源，产出文件级只读常量。INPUT 执行完毕后：
- 所有临时资源（连接、缓冲区等）卸掉
- 仅保留产出的常量供 TASK、WORKS 引用
- 之后不可再回溯 INPUT

引用方向：INPUT 可引用 TOOLS 区定义的函数（见装饰器）。

---

## 2. 区声明

```
<INPUT区>   : INPUT : { <数据源声明>+ }
```

## 3. 数据源声明模板

```
<数据源声明>    : <源关键词> : <地址>
                    [ ( <参数列表> ) ]
                    [ { <数据源声明>+ } ]
                    [ <装饰器> ]
                    [ <赋值箭头> <变量> : <类型> ]

<源关键词>  : WEBS | FILES | MEMS | ...
<地址>      : 字符串字面量（URL、文件路径、内存地址等）
```

### 3.1 地址是唯一必填项

数据源声明中，**只有 `<地址>` 是必须的**。其余部分全部可选：

```
HTTP : "https://api.com/data"          # 合法。无参数、无装饰器、无赋值。有意义但不危险
HTTP : "https://api.com/data" => X     # 加赋值，有产出
HTTP : "https://api.com/data" (timeout=30s) {   # 全套
    JSON : _ => P : record
} [decrypt] => X : bytes
```

> 无产出变量的数据源声明语法上允许，编译器和运行时不做限制，但不会产生有意义的输出。

### 3.2 各部分含义

| 部分 | 必填 | 说明 |
|------|------|------|
| `<源关键词>` | 是 | 数据来源的类型声明词 |
| `<地址>` | **是** | 资源定位。**所有源声明语义统一**——WEB 地址、文件路径、内存地址，都是地址 |
| `(参数列表)` | 否 | 源专属参数（超时、偏移、编码等） |
| `{数据源声明}` | 否 | 嵌套子数据源。内层源可通过 `_` 引用外层源的原始数据（如 `JSON : _ => PARSED`），外层装饰器处理外层数据流，内层产出为局部变量 |
| `[装饰器]` | 否 | 引用 TOOLS 函数处理数据流（默认提供一批内置装饰器，详见 TOOLS 语法） |
| `<赋值箭头>` | 否 | `=` COPY 或 `=>`/`<=` MOVE |
| `<变量> : <类型>` | 否 | 产出变量的名称与类型。不写则取数据但不绑定 |

### 3.3 装饰器在 INPUT 中的数据流

在 INPUT 语境中，装饰器的执行顺序是：

```
源协议 (HTTP/FILES/WEBS/MEMS...)  →  装饰器加工  →  产出变量
     │                                    │              │
     │  原始数据（字节流/文本/结构）        │              │
     │                                    │              │
     │                          [gzip] [decrypt] ...     │
     │                           加工后数据               │
     │                                                   │
     │                                             => RAW : bytes
```

```
INPUT :
    HTTP : "https://api.com/data" [gzip] [decrypt] => RAW : bytes
    # ① HTTP 请求拿到原始字节
    # ② gzip 解压
    # ③ decrypt 解密
    # ④ 结果绑定到 RAW
```

此行为是 INPUT 特有的——其他区域（OUT、TASK）的装饰器执行模式由各自语法定义。

## 4. 变量作用域

### 4.1 层级规则

INPUT 区按嵌套层级决定变量可见性：

| 层级 | 位置 | 可见性 |
|------|------|--------|
| 0 层 | `INPUT {}` 的直接子声明 | 产出为文件级**常量**，TASK/WORKS 可见 |
| 1+ 层 | 源声明 `{}` 内部的子声明 | 产出为**局部变量**，仅本级和同级可见，上级不可见 |

嵌套深度不设上限，每层作用域规则一致。

```
INPUT {                               # 层级 0
    HTTP : "url" {                    # ─┐
        JSON : ... => X : str         #  │ 层级 1
        JSON : ... => Y : i32         #  │ 局部变量
    } => BODY : bytes                 # ─┘ BODY 是常量（层级 0）
    
    TCP : "host:port" {               # ─┐
        # 此处可引用 X 和 Y 吗？       #  │ 不能——X、Y 在兄弟分支的嵌套内
        # 此处可引用同级声明的变量      #  │ 同一父级下平级可见
    } => STREAM : bytes               # ─┘ STREAM 是常量（层级 0）
}
```

### 4.2 变量名不可跨层引用

嵌套块内的变量对外不可见。如需在同一块内聚合多个变量，使用 JOIN（§5）。不存在变量自动"冒泡"到上级的机制。

## 5. JOIN

### 5.1 模板

```
<JOIN声明>  : JOIN : <变量名列表> <赋值箭头> <结果变量> : <类型>
```

`JOIN` 收集同一作用域下的多个变量，合并为一个新变量。JOIN 不跨作用域操作，不产生数据流。

### 5.2 合并规则

结果变量的类型标注决定合并行为：

| 类型 | 行为 | 说明 |
|------|------|------|
| 无标注 或 `dict` | 字典结构 | `{变量名: 值, 变量名: 值, ...}` |
| `list` | 值列表 | `[值, 值, 值, ...]` |
| `str` | 逗号分隔 | 各值独立转字符串，逗号拼接（`22, 434, 43` → `"22,434,43"`） |
| `int` | 求和 | 所有值加总（`22, 434, 43` → `499`） |
| `float` | 求和 | 所有值加总 |

```
INPUT {
    HTTP : "https://api.com/data" {
        JSON : _ => NAMES : list
        JSON : _ => IDS   : list
        
        JOIN : NAMES, IDS => USER_DATA : dict
        # USER_DATA = {NAMES: [...], IDS: [...]}
    } => RAW : bytes
    # RAW 是 HTTP 裸产出
    # USER_DATA 仅在内层有效，不冒泡到外层
}
```

### 5.3 非法情况

| 情况 | 说明 |
|------|------|
| JOIN 引用跨作用域变量 | 在嵌套块内引用兄弟块的局部变量 |
| JOIN 结果类型与实际值不兼容 | 声明 `int` 但含非整数值 |
| JOIN 空变量列表 | `JOIN : => x` |

## 6. 数据源关键词

数据源关键词分为三层：**基础源 → 衍生源 → 扩展源**。衍生源建立于基础源之上，扩展源视需求可选。

当前定义的数据源关键词为**最终集合**——语法核心不再新增数据源关键词。未来出现的新协议、新格式，通过以下方式解决：

| 扩展方式 | 说明 |
|----------|------|
| **用户依赖** | 第三方通过 TOOLS 函数 + 装饰器封装新协议，不侵入语言核心 |
| **内置装饰器扩充** | 官方按需新增装饰器（如 `[msgpack]`、`[avro]`），不新增源关键词 |
| **内置函数扩充** | 官方按需新增内置函数，覆盖通用数据处理需求 |

语法核心保持稳定——源关键词的完备性建立在三个基础源（WEBS / FILES / MEMS）覆盖了所有 I/O 通道的假设之上。任何新协议最终都是"通过网络 Socket 读字节"或"从文件读字节"——基础源已覆盖。

### 6.1 三个基础源

| 关键词 | 性质 | 地址格式示例 | 说明 |
|--------|------|-------------|------|
| `WEBS` | 网络基础（Socket） | `"192.168.1.1:8080"` / `"wss://stream.com"` | 最底层网络通信，裸字节流。命名 = Web + Socket 缩写，涵盖所有基于 Socket 的网络通信（TCP/HTTP/WebSocket/原始 Socket），非"Web 专用" |
| `FILES` | 文件基础 | `"/tmp/input.csv"` / `"C:\data\in.txt"` | 通读任何文件，裸字节输出，自行解析 |
| `MEMS` | 内存基础 | 引用前序常量或直接值 | 内存中的数据源，最原始的访问方式 |

### 6.2 衍生源（网络·基于 WEBS）

| 关键词 | 说明 | 必要程度 |
|--------|------|----------|
| `TCP` | 原生 TCP 通信 | 必要 |
| `HTTP` | HTTP/HTTPS（支持 HTTP/1.1、HTTP/2，前瞻 HTTP/3） | 必要 |
| `DB` | 数据库通用接口，通过连接字符串区分具体数据库 | 必要 |
| `OSS` | 对象存储协议（S3 兼容、阿里 OSS、MinIO 等） | 较常用 |

> **关于数据库**：用一个通用的 `DB` + 连接字符串，不一个数据库一个关键词。各数据库差异由驱动层处理。

### 6.3 衍生源（文件·基于 FILES）

| 关键词 | 说明 | 必要程度 |
|--------|------|----------|
| `TXT` | 纯文本文件 | 必要 |
| `CSV` | 逗号分隔值 | 必要 |
| `JSON` | 标准 JSON | 必要 |
| `JSONS` | JSON Lines（每行一个 JSON 对象） | 较常用 |
| `YAML` | YAML 格式 | 较常用 |
| `TOML` | TOML 格式 | 较常用 |
| `XML` | XML 格式 | 可选 |
| `DOCS` | Office 文档 | **不适合 INPUT**——解析办公文档超出 INPUT 职责 |
| `MD` | Markdown | **不适合 INPUT**——MD 解析更适合放在 OUTPUT 区做渲染 |

### 6.4 衍生源（内存·基于 MEMS）

| 关键词 | 说明 | 必要程度 |
|--------|------|----------|
| `MEMS` | 纯内存地址访问 | 必要 |

MEMS 是三个基础源中功能最简的：地址就是内存地址（统一语义），参数可选地指定偏移和大小。

```
MEMS : "0x7FFF1234"                             # 裸地址
MEMS : "0x7FFF1234" (offset=64, size=128)       # 切片
MEMS : "buffer_x"                               # 抽象名/句柄
```

> 内存的寻址方式（指针、偏移、句柄名）统一由地址字段表达，不多开关键词。内部细节由 MEMS 实现层处理，语法层不暴露复杂性。

### 6.5 必要程度总览

| 级别 | 含义 | 关键词 |
|------|------|--------|
| **必要** | 语言核心自带 | `WEBS` `FILES` `MEMS` `TCP` `HTTP` `DB` `TXT` `CSV` `JSON` |
| **较常用** | 推荐内置 | `JSONS` `YAML` `TOML` `OSS` |
| **可选** | 按需实现 | `XML` |

## 7. 示例

### 7.1 基础 HTTP

```
INPUT :
    HTTP : "https://api.com/data" => RAW : bytes
```

### 7.2 带参数和装饰器

```
INPUT :
    HTTP : "https://api.com/data"
        (method=GET, timeout=30s, headers={"Auth": "Bearer xxx"})
        [decrypt]
        => DATA : bytes
```

### 7.3 嵌套结构体

嵌套是纯语义组织，不存在跨层数据流。不嵌套也能达到同样效果，嵌套只为表达清晰：

```
INPUT :
    HTTP : "https://api.com/data" (method=GET, timeout=30s) {
        JSON : _ (encoding="utf8") {
            # _ = 引用父源 HTTP 的原始数据
            # JSON 解析父源数据，产出内层变量
        } => PARSED : record
    } [decrypt] => RAW : bytes
    # decrypt 装饰器处理 HTTP 字节流后绑定到 RAW
    # PARSED 是内层局部变量，不冒泡到外层
```

### 7.4 多数据源

```
INPUT :
    HTTP : "https://auth.api.com/token" => TOKEN : str
    FILES : "/tmp/batch.csv" [decompress] => CSV : bytes
    YAML : CSV (delimiter=",") => CONFIG : config
```

---

## 8. 非法情况

| 情况 | 说明 |
|------|------|
| 非源关键词出现在 `INPUT : {}` 内 | 在 INPUT 中写 `CALL` 或 `TASK` |
| 源关键词出现在 INPUT 区外 | 在 TOOLS 中写 `HTTP : ...` |
| 地址非字符串字面量 | `HTTP : 42` |
| 产出常量与已有常量同名 | 同一 INPUT 区内重复变量名 |
| 嵌套体中使用本层已消耗的数据 | MOVE 后的源不可在嵌套体中使用 |
