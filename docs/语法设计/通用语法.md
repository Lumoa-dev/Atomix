> 架构版本: v0.1 (设计阶段)
> 最后更新: 2026-07-15
> 适用范围: 所有语法域
> 配套文档: 详见 01-总纲与哲学.md

## 1. 大小写规则

Atomix 是**大小写不敏感**的语言。以下写法解析为同一标识：

```
USE    use    Use    uSe    # 同一关键词
true   TRUE   True   tRuE   # 同一字面量
```

但语言定义了一套**大小写惯例**，区分两种角色：

| 角色 | 惯例 | 示例 |
|------|------|------|
| **声明/结构词** | 全大写（大部分） | `USE` `TASK` `CALL` `INPUT` `OUT` `TOOLS` `WORKS` `FOR` `IF` `GOOUT` `ASSERT` `RAISE` `CONST` `PUB` `WAIT` `JOIN` |
| **声明/结构词** | 全小写（函数相关，历史约定） | `fn` `return` `do` |
| **值/运算符词** | 全小写 | `and` `or` `not` `true` `false` |

编译器不强制检查惯例。不遵循惯例的代码在语法上合法，但违反工程规范。

## 2. 词法规则

### 2.1 字符编码

源文件为 UTF-8 编码，无 BOM。

### 2.2 空白与符号

```
<空白>      : <空格> | <制表符> | <换行>
```

- 缩进为工程规范，非语法要求。花括号 `{}` 决定块结构，解析器不依赖缩进。
- 语句以换行或闭合花括号结束，**无分号**。

### 2.3 注释

```
<行注释>    : # <任意内容直至换行>
<块注释>    : #! <任意内容> !#
```

- 块注释不可嵌套
- 注释等同于空白，对解析无影响
- `#` 在字符串字面量内为普通字符，不视为注释起始

## 3. 标识符

```
<标识符>    : <起始> <延续>*

<起始>      : a-z | A-Z | _
<延续>      : a-z | A-Z | 0-9 | _
```

- 大小写不敏感
- 标识符不可与关键词同名（不区分大小写）
- `$` 为保留特殊标识符——唯一的单字符非字母标识符，详见 §9

## 4. 关键词

### 4.1 语法域

关键词按适用范围分为两层：

| 层 | 说明 |
|----|------|
| **通用关键词** | 在所有语法域中合法 |
| **域关键词** | 仅在所属语法域中合法，跨域使用为语法错误 |

此即 **语法域（Syntax Domain）**——同一语言内，不同区域（TOOLS / INPUT / TASK / WORKS / OUT）各自拥有独立的语法子集。通用语法仅定义跨所有域合法的结构。

### 4.2 通用关键词

```
<通用关键词>    : USE
                | true | false
                | and | or | not
```

| 关键词 | 惯例 | 引入结构 |
|--------|------|----------|
| `USE` | 大写 | 外部依赖声明（文件头） |
| `true` `false` | 小写 | 布尔字面量 |
| `and` `or` `not` | 小写 | 逻辑运算符 |
| `fn` | 小写 | 函数/方法定义（TOOLS、WORKS 域） |
| `ASSERT` | 大写 | 断言（通用） |
| `RAISE` | 大写 | 抛异常（通用） |
| `return` | 小写 | 函数返回值（通用） |
| `do` | 小写 | 匿名函数（通用） |

### 4.3 非法情况

| 情况 | 说明 |
|------|------|
| 关键词做标识符 | `use = 42`（`use` 是关键词） |
| 域关键词跨域使用 | 在 INPUT 区写 `call : foo()` 为语法错误 |

## 5. 字面量

```
<整数>      : <十进制> | <十六进制> | <二进制> | <八进制>
<浮点数>    : <十进制整数> . <十进制数字> [ e <指数> ]
            | <十进制整数> e <指数>
<字符串>    : " <字符序列> "
<F字符串>   : f" <字符序列> [ <插值> ] "
<布尔值>    : true | false

<十进制>    : 0 | [1-9] [0-9]*
<十六进制>  : 0x <十六进制数字>+
<二进制>    : 0b [01]+
<八进制>    : 0o [0-7]+

<指数>      : [+|-] <十进制数字>+
<转义>      : \n | \t | \r | \\ | \" | \x <十六进制数字><十六进制数字>
<插值>      : { <值表达式> }
```

### 5.1 F-字符串

F-字符串以 `f"` 开头，支持在字符串中嵌入求值表达式。格式为 `{expression}`：

```
name = "atomix"
version = 42
greeting = f"this is {name} v{version}"
# 求值结果: "this is atomix v42"
```

插值表达式 `{}` 内可以放任意值表达式：

```
result = f"computed: {a + b}"
```

### 5.2 非法情况

| 情况 | 说明 |
|------|------|
| 整数前导零（非 0） | `007`（`0` 本身合法） |
| 空进制前缀 | `0x`（无后续数字） |
| 浮点数无整数部分 | `.5`（需写作 `0.5`） |
| 字符串未闭包 | `"hello` |
| 字符串含裸换行 | 字符串内换行必须用 `\n` |
| 未知转义 | `\q` `\xGG` |

## 7. 值表达式

```
<值表达式>  : <字面量>
            | <标识符>
            | <一元运算>
            | <二元运算>
            | ( <值表达式> )
```

### 7.1 一元运算

```
<一元运算>  : <一元运算符> <值表达式>

<一元运算符>    : -
                | not
                | ~
```

| 运算符 | 操作数要求 | 结果 |
|--------|-----------|------|
| `-` | 数值 | 算术取负 |
| `not` | 布尔 | 逻辑非 |
| `~` | 整数 | 按位取反 |

### 7.2 二元运算

```
<二元运算>  : <值表达式> <二元运算符> <值表达式>

<二元运算符>    : +  | -  | *  | /  | %
                | == | != | <  | >  | <= | >=
                | and | or
                | &  | |  | ^  | << | >>
```

优先级（从高到低）：

```
*   /   %                左结合    乘除取余
+   -                    左结合    加减
<<  >>                   左结合    移位
&                        左结合    按位与
^                        左结合    按位异或
|                        左结合    按位或
==  !=  <  >  <=  >=    左结合    比较（返回布尔）
and or                  左结合    逻辑（短路求值）
```

### 7.3 非法情况

| 情况 | 说明 |
|------|------|
| 数字 + 布尔 | `42 + true` |
| `and`/`or` 操作非布尔 | `42 and 0` |
| 位运算操作非整数 | `true << 2` |
| `not` 操作非布尔 | `not 42` |
| `-` 操作非数值 | `-"hello"` |
| 链式比较 | `1 < x < 5`（需写作 `1 < x and x < 5`） |

## 8. 赋值与值移动

Atomix 有三种值传递符号，由内存动作语义区分：

| 符号 | 语义 | 内存动作 | 说明 |
|------|------|----------|------|
| `=` | 绑定赋值 | **COPY** | 值复制一份给左侧绑定 |
| `=>` | 前向输送 | **MOVE** | 值从左到右移动，源失效 |
| `<=` | 反向回引 | **MOVE** | 值从右到左移动，源失效 |

`=>` 和 `<=` 都是 MOVE，提供两个方向是为了解决方向偏好问题——同一段数据流，有的人习惯从左往右写（`=>`），有的人习惯从右往左写（`<=`）。两者并存，按语境选用。

> **MOVE 是内存级操作，不影响外部源。** 文件还在磁盘上，HTTP 服务端的资源还在那。MOVE 只是把内存里的数据挪走、原缓冲区清理掉。重新再来的话需要重新寻址、重新加载、重新请求。
>
> `=` COPY 则保留一份在内存原位置，后续可再次引用。

### 8.1 `=` 绑定赋值

```
<绑定赋值>  : <标识符> = <值表达式>
```

读作"绑定为"。COPY 语义——右侧值完整复制一份给左侧标识符：

```
x = 42
name = "atomix"
transformed = CALL : process(INPUT : RAW)
```

`=` 左侧必须为标识符，右侧必须为值表达式。

### 8.2 `=>` 前向移动

```
<前向移动>  : <左值> => <右值>
```

读作"流向"。MOVE 语义——数据从左侧移至右侧，左侧源失效：

```
INPUT : HTTP : "https://api.com/data" => RAW : bytes
```

`=>` 在 INPUT、TASK、OUT 三个语法域中出现，各域角色详见各自文档。

### 8.3 `<=` 反向移动

```
<反向移动>  : <左值> <= <右值>
```

读作"回引"。MOVE 语义——数据从右侧移至左侧，右侧源失效：

```
TASK :
    audit = CALL : log(result)
    seq <= audit       # audit 的值移入 seq，audit 不可再用
```

### 8.4 非法情况

| 情况 | 说明 |
|------|------|
| `=` 左侧非标识符 | `a + b = 42` |
| `=>` 左侧为字面量 | `42 => x` |
| `<=` 左侧为字面量 | `"hello" <= x` |
| 箭头串行 | `a => b => c`（箭头不可链式） |
| MOVE 后读源 | `=>`/`<=` 的源端在操作后不可再引用 |

## 9. `$` 管道变量

`$` 是一个特殊的内置变量，属于 `<标识符>` 的语法例外——它是唯一的单字符非字母标识符，在语法中既是**操作符**又是**隐式容器**。

### 9.1 语法定位

`$` 在词法层面属于 `<标识符>` 的保留例外（见 §3）。`$[key]` 的 `[key]` 访问语法与列表/字典访问（通用语法 §6、类型系统 §6）共享同一 `[ ]` 后缀操作符——`$` 作为隐式结构体容器，`$[key]` 等价于对 `$` 执行键值访问。

### 9.2 管道触发

在 CALL 或 WAIT 后写 `$`，触发管道模式——调用/派发结果以 KV 结构存入 `$`：

```
CALL fetch_data() $        # 结果存入 $，如 ${a: val1, b: val2}
CALL process() $           # 滚动替换，$ 内容被新结果覆盖
WAIT DataPipeline() $      # WAIT 同样支持管道
```

### 9.3 管道引用

后续调用中通过 `$[key]` 取出当前 `$` 中的值：

```
CALL fetch_data(raw1, raw2) $     # 返回 {a, b, c}
CALL process($[a], $[b], $[c]) $  # 取 a b c，返回 {d, e}
CALL save($[d], $[e]) => result   # 取 d e，绑定到 result
```

### 9.4 生命周期

`$` 的值**滚动替换**——每次用 `$` 触发管道，旧值被新值完全覆盖：

```
CALL step1() $      # $ = {a: 1}
CALL step2($[a]) $  # $ = {b: 2}  （此时旧值 a 已丢失）
CALL step3($[b])    # ✅ 取当前的 b
CALL step3($[a])    # ❌ a 已不存在，step2 的 $ 覆盖了它
```

生命周期规则：
- **开始**：CALL 尾部出现 `$`，结果写入 `$`
- **结束**：下一次 CALL 尾部出现 `$`，旧值被覆盖
- 若后续无新的 `$` 出现，当前 `$` 保持到最后

### 9.5 非法情况

| 情况 | 说明 |
|------|------|
| `$` 用于非调用位置 | `x = 42 $` |
| `$[key]` 但 key 不存在 | 引用当前 `$` 中未返回的字段名 |
| `$` 未初始化时引用 | 尚未有任何调用触发管道时 |

## 10. 变量与声明

```
<声明>      : [ <声明关键词> ] <标识符> : <类型> <运算符> <值表达式>
```

### 10.1 无关键词声明

不写关键词就是普通变量。该分配分配，该回收回收，无附加行为：

```
x : int = 42
name : str = "atomix"
total : f64 = a + b * 2
```

类型标注 `: <类型>` 不可省略。

### 10.2 关键词声明

加关键词就是加行为：

```
CONST pi : f64 = 3.14159    # 常量，运行时不可变
GOOUT result : int = CALL : compute()
```

各关键词的具体附加行为由所属语法域定义：

| 关键词 | 适用范围 | 附加行为 |
|--------|----------|----------|
| `CONST` | 通用 | 声明为常量，绑定后不可修改 |
| `GOOUT` | 任意区（除 INPUT） | 声明为对外产出变量，对 OUT 区可见，交付即终结 |

> `GOOUT` 不限定在 TASK 区。任何区域（TOOLS、WORKS、TASK 等）都可以用 `GOOUT` 标注变量，标注后即可被 OUT 区引用并写出。INPUT 区使用 `GOOUT` 无意义，语法不禁止但工程规范不建议。
>
> 域专属声明词详见各区语法文档，声明变量模板详见 §9.1。

### 10.3 非法情况

| 情况 | 说明 |
|------|------|
| CONST 后二次赋值 | `CONST x : int = 1` 后 `x = 2` |
| 类型省略 | `x = 42`（缺少 `: <类型>`） |
| 域关键词跨域使用 | 在 INPUT 区写 `OUT x : int = ...` |

## 11. 作用域规则

### 11.1 块作用域

花括号 `{}` 创建新作用域。内层可访问外层变量，外层不可访问内层：

```
x : int = 1
{
    y : int = 2    # y 仅在块内可见
}
# y 在此处不可访问
```

### 11.2 变量遮蔽

内层作用域可声明与外层同名的变量，外层变量被遮蔽：

```
x : int = 1
{
    x : int = 2    # 遮蔽外层 x
    # 此处 x = 2
}
# 此处 x = 1（外层恢复）
```

### 11.3 IF / FOR 作用域

条件体和循环体各自创建子作用域：

```
IF a > 0 {
    tmp : int = a * 2   # tmp 仅在 IF 块内
}
# tmp 不可访问

FOR i < 10 {
    acc : int = acc + i  # 每次循环独立作用域
}
```

### 11.4 TASK 有效入口

TASK 是文件的**实际执行入口**。只有从 TASK 可达的代码才是"有效代码"：

```
TOOLS {
    fn used() { ... }       # ✅ 被 TASK 调用 → 有效
    fn unused() { ... }     # ❌ 从未被 TASK 调用 → 无效（编译器可告警）
}

TASK {
    CALL used()
}
```

匿名函数定义了一个作用域闭包，但同样只在实际执行时才生效：

```
do (x : int) { x * 2 }     # 当场执行，有效
fn make() { do (x) { x } } # do 在被调用时才执行
```

### 11.5 跨区可见性

| 声明位置 | 可见范围 |
|----------|----------|
| 文件级常量（INPUT） | 全局（TASK、WORKS、OUT、TOOLS） |
| `GOOUT` 变量 | 对 OUT 区可见，交付即终结 |
| TOOLS 函数 | 全局注册，装饰器不需前缀 |
| 区内局部变量 | 仅所属区内 |
| 块内局部变量 | 仅所属块内 |

### 11.6 非法情况

| 情况 | 说明 |
|------|------|
| 访问未声明变量 | `x = y` 但 y 未在当前作用域声明 |
| 跨区引用局部变量 | 在 TASK 区访问 INPUT 区的块内局部变量 |
| 在变量声明前使用 | 声明语句之前的代码行中引用（无 hoisting） |

## 12. 断言

```
<ASSERT声明> : ASSERT <值表达式> [ , <字符串> ]
```

断言在**所有语法域中**都可使用。条件为 `false` 时中断执行并抛出异常，可选附带消息：

```
ASSERT x > 0
ASSERT data != "", "data should not be empty"
ASSERT result == expected, f"expected {expected}, got {result}"
```

断言不做类型限制——任何值表达式都可以作为条件。

### 12.1 非法情况

| 情况 | 说明 |
|------|------|
| 条件非布尔值 | 运行时求值后检查（不强求编译期类型检查） |

---

## 13. RAISE

```
<RAISE声明> : RAISE <值表达式> [ , <值表达式> ]
```

触发异常。任何语法域都可使用——条件自己套，RAISE 只管抛：

```
RAISE IOError
RAISE "something went wrong"
RAISE TimeoutError, "request timed out after 10s"
```

### 13.1 非法情况

| 情况 | 说明 |
|------|------|
| RAISE 不接任何参数 | `RAISE`（必须指定抛什么） |

---

## 14. return

```
<RETURN声明> : return [ <值表达式> ]
```

函数内返回结果。不写表达式时返回空值：

```
fn add(x : int, y : int) : int {
    return x + y
}

fn log_only(msg : str) {
    print(msg)
    return          # 无返回值
}
```

函数内最后一条表达式省略 `return` 时，自动作为返回值（隐式返回）：

```
fn add(x : int, y : int) : int { x + y }    # 隐式 return x + y
```

### 14.1 非法情况

| 情况 | 说明 |
|------|------|
| `return` 出现在函数外 | 在 TASK 或 INPUT 中写 `return` |
| 返回类型不匹配 | 函数声明 `: int` 但 `return true` |

## 15. BREAK / CONTINUE

`BREAK` 和 `CONTINUE` 是循环体内的控制流跳转，**必须出现在 `FOR` 循环块内**。出现在循环外为语法错误。

两者均带条件——条件为 `true` 时才触发跳转。不带条件则等价于条件恒为 `true`。

### 15.1 BREAK

```
<BREAK声明> : BREAK [ <值表达式> ]
```

条件为 `true`（或不写条件）时，立即终止当前最近一层 `FOR` 循环，跳转到循环块之后的第一条语句。

```
FOR i < 100 {
    CALL process(items[i])
    BREAK items[i] == 0       # 遇到 0 就终止
    i = i + 1
}
```

### 15.2 CONTINUE

```
<CONTINUE声明> : CONTINUE [ <值表达式> ]
```

条件为 `true`（或不写条件）时，跳过当前迭代剩余代码，直接进入下一次循环条件求值。

```
FOR i < 100 {
    i = i + 1
    CONTINUE items[i] == 0    # 跳过 0 值的处理
    CALL process(items[i])
}
```

### 15.3 非法情况

| 情况 | 说明 |
|------|------|
| `BREAK` 出现在 `FOR` 循环外 | 无循环可跳出 |
| `CONTINUE` 出现在 `FOR` 循环外 | 无循环可继续 |
| 条件非布尔值 | 运行时求值后检查 |

## 16. 函数定义

函数定义只在 TOOLS 和 WORKS 语法域中出现（INPUT、TASK、OUT 不可定义函数）。语法在此统一定义，各区直接引用。

### 16.1 模板

```
<函数定义>  : fn <名称> [ <泛型参数> ] ( <参数列表> ) [ : <返回类型> ] { <代码体> }

<泛型参数>  : < <类型参数> [ , <类型参数> ]* >
<类型参数>  : <标识符>
<参数列表>  : <参数> [ , <参数> ]*
<参数>      : <标识符> : <类型> [ = <值表达式> ]
```

各部分含义与通用语法保持一致——参数类型标注、返回值类型标注均使用 `: type` 语法（详见 §6 类型标注）：

```
fn add(x : int, y : int) : int {
    x + y
}

fn get_name() : str {
    self.NAME
}

fn log() {                  # 无返回值，不写返回类型
    print("done")
}
```

### 16.1.1 默认参数值

参数可附带默认值（`= <值表达式>`）。调用时若未提供该参数，使用默认值：

```
fn greet(name : str = "world") : str {
    "hello " + name
}

fn fetch(url : str, timeout : int = 30, retry : int = 3) : bytes {
    ...
}

greet()                  # name = "world"
greet("atomix")          # name = "atomix"
fetch("https://...")     # timeout = 30, retry = 3
fetch("https://...", 60) # timeout = 60, retry = 3
```

**规则：**

- 有默认值的参数必须排在无默认值参数**之后**
- 默认值表达式在**调用点**求值（非定义时），可引用前方已绑定的参数
- 调用时按位置传递，不可跳过中间参数（`fetch(url, , 5)` 非法）

```
fn example(a : int, b : int = a * 2, c : int = b + 1) { ... }
# b 默认值引用 a，c 默认值引用 b — 合法
```

### 16.2 泛型函数

`<` `>` 声明类型参数。类型参数可在参数列表和返回类型中作为类型使用：

```
fn identity<T>(x : T) : T { x }
fn first<T>(list : list[T]) : T { list[0] }
fn pair<A, B>(a : A, b : B) : tuple(A, B) { (a, b) }
```

调用时编译器根据实参类型推断类型参数，并**单态化**为具体类型版本：

```
x = identity(42)          # T → int，生成 identity_int
y = identity("hello")     # T → str，生成 identity_str
p = pair(1, true)          # A → int, B → bool，生成 pair_int_bool
```

也可显式指定类型参数（当编译器无法推断时）：

```
x = identity<int>(42)
```

**单态化规则：**

- 编译器为每种唯一的类型参数组合生成一份独立函数体
- 单态化后的函数与手写具体类型版本的运行时性能完全一致
- 不对类型参数做运行时表示——无 boxing、无虚表、无类型标签
- 未使用的泛型函数不参与编译（零开销抽象）

**非法情况：**

| 情况 | 说明 |
|------|------|
| 类型参数未在签名中使用 | `fn f<T>(x : int) : int`（`T` 未出现在参数或返回类型中） |
| 对类型参数做不支持的操作 | `fn add<T>(a : T, b : T) : T { a + b }`（`T` 不一定是数值类型——但编译器按调用点单态化后检查，此处不提前报错） |

### 16.3 公开声明

方法默认私有（仅 WORKS 内部可见）。公开方法使用 `PUB` 声明（详见 WORKS 语法 §6.3）：

```
PUB fn get_name()           # 先声明
fn get_name() { ... }       # 后定义

# 或内联
PUB fn get_name() : str {
    self.NAME
}
```

公开泛型方法同样适用：

```
PUB fn map<A, B>(input : list[A], f : fn(A) : B) : list[B]
```

### 16.4 非法情况

| 情况 | 说明 |
|------|------|
| 函数定义出现在禁止区 | 在 INPUT / TASK / OUT 中写 `fn` |
| 返回类型与实际返回值不匹配 | 声明 `: int` 但返回非数值 |
| 公开声明后无对应定义 | `PUB fn x()` 后无 `fn x() {}` |
| 类型参数名与已有类型重名 | `fn first<list>(...)`（`list` 是内置类型名） |
| 有默认值的参数排在无默认值参数之前 | `fn f(a : int = 1, b : int)` — `b` 无默认值但排在 `a` 之后 |
| 调用时跳过中间参数 | `fetch(url, , 5)` — 不可跳跃传参 |

## 17. 匿名函数

```
<匿名函数> : do ( <参数列表> ) [ : <返回类型> ] { <代码体> }
```

匿名函数没有名字，**当场写当场执行**。参数、类型标注、代码体的语法与函数定义完全一致：

```
do (x : int, y : int) : int {
    x + y
}

result = do (a : str, b : str) : str {
    a + " " + b
}
```

`do` 可以出现在任何值表达式的位置——作为参数传递、赋值给变量、或直接求值：

```
CALL process(do (x : int) { x * 2 })
```

### 17.1 非法情况

| 情况 | 说明 |
|------|------|
| `do` 没有参数和体 | `do` 单独出现 |
| 匿名函数内使用 `PUB` | `PUB do () {}`（匿名函数没有可见性概念） |

## 18. 装饰器

```
<装饰器>    : [ <标识符> ]
```

装饰器挂在 `=>` 之前，数据流经时先经装饰器函数处理：

```
HTTP : "url" [compress] => RAW : bytes
```

- 装饰器标识符引用 TOOLS 区定义的函数
- 多装饰器可串联：`[gzip] [encrypt] =>`
- 装饰器紧跟 `=>` 之前，不可放在其后

### 18.1 非法情况

| 情况 | 说明 |
|------|------|
| 装饰器引用未定义函数 | `[undefined_fn]` |
| 装饰器位置错误 | `=> [compress]` |
| 装饰器内非标识符 | `[a + b]` |

## 19. 尾随关键词

尾随关键词是独立于前置关键词的另一套词表，**仅出现在调用之后**，为调用附加行为。

```
<尾随>      : <尾随关键词> [ <块> ]

<尾随关键词> : TRY | ...
```

- 前置关键词（`CALL`、`OUT` 等）不可当尾随词
- 尾随关键词不可当前置词
- 两套表互不交叉

### 19.1 `TRY`

`TRY` 为调用附加异常处理作用域：

```
<TRY用法>   : <调用式> TRY [ <值表达式> ] [ { <异常处理体> } ]
```

`<值表达式>` 为匹配条件（详见 §7），其中可使用 TRY 作用域内置的只读变量。不带条件时捕获所有异常：

```
CALL : process(INPUT : RAW) TRY {
    # 捕获所有异常
}
```

带条件时仅捕获匹配的异常：

```
CALL : fetch_data() TRY ISTIMEOUT == 10s {
    # 仅处理超时
}
CALL : parse() TRY ISERROR is TypeError {
    # 仅处理类型错误
}
CALL : validate() TRY ISERROR is TypeError or ISTIMEOUT > 5s {
    # 复合条件
}
```

TRY 块内部可使用以下内置变量：

| 内置变量 | 含义 | 类型 |
|--------|------|------|
| `ISERROR` | 当前捕获的异常对象 | 错误类型 |
| `ISTIMEOUT` | 当前超时时间值 | 时间量 |
| `ISBIGSIZE` | 当前数据大小值 | 尺寸量 |
| `ISWARNING` | 当前警告级别 | 警告级别值 |

```
CALL : fetch_data() TRY ISTIMEOUT == 10s {
    print(ISTIMEOUT)    # 实际超时值
    print(ISERROR)      # 实际错误对象
}
```

### 19.2 链式异常处理

不带块的 `TRY` 自动挂到**后方最近一个带块且条件匹配的 `TRY`** 上：

```
CALL : fetch() TRY ISTIMEOUT == 10s     # 这三个 TRY 不带块
CALL : parse() TRY ISTIMEOUT == 10s     # 共享最后一个 TRY {} 的处理体
CALL : save() TRY ISTIMEOUT == 10s
CALL : cleanup() TRY ISTIMEOUT == 10s { # 这个带块，接管上面三个的异常
    # 上面三个调用抛出超时异常，全部落在这里
}
```

所有共享同一 `TRY {}` 的调用，其异常处理行为完全一致。链中各调用的条件必须一致，否则不可共享。

### 19.3 非法情况

| 情况 | 说明 |
|------|------|
| 尾随词用于非调用场景 | `x = 42 TRY` |
| 链式异常处理链无终点 | `CALL : a() TRY` 后无 `TRY {}` 接住 |
| 尾随词当前置词 | `TRY : x` |
| 链式异常处理链条件不一致 | 各 `TRY` 条件不同却共享同一 `TRY {}` |

## 20. USE 声明

```
<USE声明>   : USE : <字符串>
```

USE 出现在文件头部，不属于任何语法域。每行一个：

```
USE : "http"
USE : "fs"
```

### 20.1 非法情况

| 情况 | 说明 |
|------|------|
| USE 出现在域内部 | `TASK : { USE : "x" }` |
| USE 重复 | 两次 `USE : "http"` |
| USE 空字符串 | `USE : ""` |

## 21. 跨域引用

```
<跨域引用>  : <域名> :: <标识符>
```

跨域引用沿数据流方向：

```
TOOLS :: compress       # 引用 TOOLS 区函数
INPUT :: RAW            # 引用 INPUT 区常量
WORKS :: data_pipeline  # 引用 WORKS 区模板
```

引用方向约束：

```mermaid
flowchart LR
    TOOLS --> INPUT --> WORKS --> TASK --> OUT
```

| 源域 | 可引用 |
|------|--------|
| TASK | TOOLS、INPUT、WORKS |
| OUT | TOOLS、TASK |
| INPUT | TOOLS |
| WORKS | TOOLS |

逆向引用（如 INPUT 引用 TASK）为非法。

### 21.1 非法情况

| 情况 | 说明 |
|------|------|
| 逆向引用 | 在 INPUT 区写 `TASK :: x` |
| 引用不存在域名 | `VOID :: x` |
| 引用域内未声明名称 | `TOOLS :: undefined_fn` |
