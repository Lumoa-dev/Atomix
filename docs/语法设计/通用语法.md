## 组合约束总表（本层规则）

本层是三层中最严格的一层。所有基元语法**顺序固定**，不接受变体。

| 组合 | 合法？ | 说明 |
|------|--------|------|
| `<类型> <变量>`（如 `int count`） | ✅ | 类型声明，创建变量 |
| `<关键字> : <变量> [: <类型>]`（如 `OUT : result`） | ✅ | 关键字声明，创建变量，类型可选 |
| `... => <变量> : <类型>`（箭头后标注） | ✅ | 类型标注附着在数据流动上 |
| `<变量> : <类型>` 独立一行 | ❌ | 类型标注不能独立出现，无数据变化来源 |
| `INT : a : int` | ❌ | 两种声明风格混合，非法 |
| 注释嵌套（`#! ... !#` 内含 `#!`） | ❌ | 块注释不可嵌套 |
| `WAIT <句柄> = <文件>(<参数>) = <变量>` | ✅ | WAIT 复制模式（详见第 5 节） |
| `WAIT <句柄> = <文件>(<参数>) => <变量>` | ✅ | WAIT 移动模式（详见第 5 节） |
| `WAIT` 中 `=` 与 `=>` 同时出现 | ❌ | 互斥，二选一 |

## 1. 词法规则

### 1.1 注释

| 形式 | 说明 |
|------|------|
| `#` | 单行注释，从 `#` 到行尾 |
| `#! ... !#` | 块注释，不可嵌套 |

### 1.2 标识符与关键字

标识符：以字母或下划线开头，后续可包含字母、数字、下划线。

```
标识符 ::= [a-zA-Z_][a-zA-Z0-9_]*
```

关键字（保留，不可用作标识符）：见第 5 节关键字总表。

### 1.3 字面量

| 类型 | 格式 | 示例 |
|------|------|------|
| 整数字面量 | 十进制 / `0x` 十六进制 | `42`, `0xFF` |
| 浮点字面量 | 带小数点 | `3.14` |
| 字符串字面量 | 双引号包裹 | `"hello"` |
| 布尔字面量 | `true` / `false` | `true` |
| 空值字面量 | `null` | `null` |
| 列表字面量 | `[<值>, ...]` | `[1, 2, 3]`、`["a", "b"]` |
| 字典字面量 | `{<键>: <值>, ...}` | `{"k": "v"}`、`{1: "a"}` |

#### 字符串转义

| 转义 | 含义 |
|------|------|
| `\n` | 换行 |
| `\t` | 制表符 |
| `\r` | 回车 |
| `\\` | 反斜杠本身 |
| `\"` | 双引号 |
| `\0` | 空字符 |
| `\xNN` | 十六进制字节（如 `\x1B`）|

### 1.4 大小写规则

大小写不敏感。推荐约定：
- 关键字使用大写（`HTTP`、`CALL`、`IF`、`FOR`、`JSON` 等）
- 变量名按上下文风格（snake_case / camelCase 均可）

## 2. 变量系统

Atomix 中创建变量有三种方式，互斥且无歧义。三种方式可在文件中混合使用。

### 2.1 类型声明（类 C 风格）

```
<类型> <变量>
```

| 段 | 必填 | 说明 |
|----|------|------|
| `<类型>` | ✅ | 类型关键字，如 `int`、`string`、`bytes`、`list` 等 |
| `<变量>` | ✅ | 变量名 |

类型关键字在前，直接创建变量。类型由前面的关键字决定。

**示例：**
```
int count
string name
bytes raw
list items
```

### 2.2 隐式声明（Python 风格）

```
<变量> = <初始值>
```

| 段 | 必填 | 说明 |
|----|------|------|
| `<变量>` | ✅ | 变量名 |
| `=` | ✅ | 赋值操作 |
| `<初始值>` | ✅ | 表达式，类型作为推导源头 |

变量名在前，不写类型关键字。类型由右侧初始值的**源头类型**推导而来。

**示例：**
```
count = 42               // 推导为 int
name = "atomix"          // 推导为 string
items = [1, 2, 3]        // 推导为 list[int]
config = {"k": "v"}      // 推导为 dict[string, string]
```

### 2.3 注解声明（Python 注解风格）

```
<变量> : <类型> = <初始值>
```

| 段 | 必填 | 说明 |
|----|------|------|
| `<变量>` | ✅ | 变量名 |
| `:<类型>` | ✅ | 类型标注，显式声明类型 |
| `=` | ✅ | 赋值操作 |
| `<初始值>` | ✅ | 表达式，类型必须匹配标注 |

变量名在前，显式标注类型，然后赋值。类型以标注为准，初始值必须可转换为该类型。

**示例：**
```
name: string = "atomix"        // 显式标注 string
count: int = 42                // 显式标注 int
items: list[int] = [1, 2, 3]   // 显式标注 list[int]

// 类型不匹配
count: int = "hello"           // ❌ 初始值类型不匹配
```

**与隐式声明的区别：**

| 特点 | 隐式声明 `x = 42` | 注解声明 `x: int = 42` |
|------|-------------------|----------------------|
| 类型来源 | 从初始值推导 | 以标注为准 |
| 类型检查 | 推导类型必须一致 | 标注类型 + 运行时值类型一致 |
| 适用场景 | 局部变量，类型一目了然 | 边界/接口，强调类型约束 |

### 2.4 关键字声明（DSL 风格）

```
<关键字> : <变量> [: <类型>]
```

| 段 | 必填 | 说明 |
|----|------|------|
| `<关键字>` | ✅ | DSL 声明关键字，如 `OUT` 等 |
| `<变量>` | ✅ | 变量名 |
| `<类型>` | ❌ | 数据类型，省略则由编译器推导 |

关键字在前，创建变量。类型可选，省略则由编译器从关键字自带类型推导。

**示例：**
```
OUT : result
OUT : batch_data : list
```

### 2.5 类型标注

```
<变量> : <类型>
```

类型标注**不是一条独立语句**。它不能凭空出现，必须附着在有数据流动的上下文中——跟在箭头 `=>` / `<=` 之后：

```
... => raw : bytes       // ✅ 数据流入 raw 时附加类型标注
raw : bytes <= ...       // ✅ 反向箭头同理
JOIN : x, y => acc : list   // ✅ 聚合结果附带类型
```

| 场景 | 结果 |
|------|------|
| 变量已存在 + 箭头上下文 `=> 变量 : 类型` | ✅ 合法，标注/变更类型 |
| 变量不存在 + 箭头上下文 `=> 变量 : 类型` | ❌ 非法，变量必须先创建 |
| 单独一行 `raw : bytes` | ❌ 非法，无数据变化来源，标注无意义 |

### 2.6 互斥规则

- 同一变量只能用一种方式创建
- 四种声明方式对同一变量**互斥**——不能对同一个变量混用两种声明
- 不允许 `INT : a : int` 这种混合写法——`INT` 不是声明关键字，`int` 也不是关键字声明前段

### 2.7 变量遮蔽规则

| 规则 | 说明 | 示例 |
|------|------|------|
| **嵌套块可遮蔽** | 内层作用域可以声明与外层同名的变量，内层生效期间外层不可见 | `x = 1; IF : (true) { x = 2 }` → 块内 x=2，块外 x=1 |
| **同一层级不可重复声明** | 同一作用域内不可声明同名变量 | `x = 1; x = 2` ❌ 同一层级重复声明 |
| **遮蔽生命周期** | 内层退出后，外层变量恢复可见 | 块/循环体结束后，遮蔽关系自动解除 |
| **函数参数不可与函数内变量同名** | 参数名在函数体中是保留标识符 | `fn foo(int x) { x = 2 }` → 这是赋值，不是遮蔽声明 |

**遮蔽与赋值的区别：**

```
int x = 1                // 声明 x
IF : (true) {
    int x = 2            // ✅ 遮蔽：新的 int x，块内生效
    x = 3                // ✅ 赋值：修改的是被遮蔽的这个 x
}
// 此处 x 恢复为外层 x（值仍为 1）

// 对比 IF 体内的普通赋值（无遮蔽——没有 int 前缀）：
int x = 1
IF : (true) {
    x = 2                // ✅ 赋值：修改的是外层 x
}
// 此处 x 的值已变为 2
```

### 2.8 解构赋值

从列表/字典中一次性提取多个值到变量。

#### 模板

```
// 列表解构
[<元素1>, <元素2>, ... [*<剩余变量>]] = <表达式>
<元素> ::= <变量> | <子列表模式> | <子字典模式>

// 字典解构
{<键名>: <变量>, ...} = <表达式>
{<变量>, ...} = <表达式>                  // 简写：变量名即键名
```

| 段 | 必填 | 说明 |
|----|------|------|
| `<元素1>, <元素2>` | ❌ | 按位置匹配的变量，数量可少于列表长度（多余丢弃） |
| `*<剩余变量>` | ❌ | 捕获剩余元素（仅限列表解构，只能出现在末尾） |
| `<子模式>` | ❌ | 支持嵌套：`[a, [b, c]]` 或 `{user: {name}}` |
| `<键名>` | ✅ | 字典中的键（字符串） |
| `<变量>` | ✅ | 接收值的变量 |
| `<表达式>` | ✅ | 右侧的列表/字典值 |

#### 禁止规则

| 写法 | 原因 |
|------|------|
| `{name} = <列表>` | 字典模式不可解列表 |
| `[a, b] = <字典>` | 列表模式不可解字典 |

#### 示例

```
[a, b] = [1, 2, 3]                     // a=1, b=2（3丢弃）
[x, *rest] = [1, 2, 3, 4]              // x=1, rest=[2,3,4]
{name, age} = {"name": "atomix", "age": 1}     // 简写
{user: {name}} = data                             // 嵌套
```

## 3. 值表达式

### 3.1 BNF

```
<值> ::= <变量>
       | INPUT : <常量名>
       | <字面量>
       | U[<变量名>]
       | <函数>(<参数>)
       | <匿名函数>
```

| 形式 | 示例 | 说明 |
|------|------|------|
| `<变量>` | `user_obj`、`config` | 本地变量引用 |
| `INPUT : <常量>` | `INPUT : APP_CFG` | INPUT 区定义的文件常量 |
| `<字面量>` | `42`、`"hello"`、`true` | 字符串/数字/布尔 |
| `U[<变量名>]` | `U[profile_data]` | 管道传参字典的值 |
| `<函数>(<参数>)` | `parse_user(raw)` | 函数调用（值上下文中） |

### 3.2 二元运算

```
<值> <逻辑符> <值>
<值> <运算符> <值>
<值> <位运算符> <值>
```

**逻辑符：** `==` `!=` `<` `>` `<=` `>=`

**类型检测：** `is`（类型/子类型判断，如 `ISERROR is TypeError`）

**算术运算符：** `+` `-` `*` `/` `%`

**位运算符：** `&` `|` `^` `<<` `>>`

**一元运算符：** `~`（按位取反） `-`（取负） `!`（逻辑非）

### 3.3 运算符优先级（从高到低）

| 优先级 | 运算符 | 结合性 |
|--------|--------|--------|
| 1 (最高) | `()` 函数调用、`U[]` 字典引用 | 左→右 |
| 2 | `~` `!` `-`（一元前缀） | 右→左 |
| 3 | `*` `/` `%` | 左→右 |
| 4 | `+` `-` | 左→右 |
| 5 | `<<` `>>` | 左→右 |
| 6 | `&` | 左→右 |
| 7 | `^` | 左→右 |
| 8 | `\|` | 左→右 |
| 9 | `&&` | 左→右 |
| 10 | `\|\|` | 左→右 |
| 11 (最低) | `==` `!=` `<` `>` `<=` `>=` `is` | 左→右 |

### 3.4 短路求值

逻辑运算符 `&&`（逻辑与）和 `||`（逻辑或）采用**短路求值**策略：

| 表达式 | 行为 |
|--------|------|
| `a && b` | 先求值 `a`。若 `a` 为 `false`，不再求值 `b`，结果为 `false` |
| `a \|\| b` | 先求值 `a`。若 `a` 为 `true`，不再求值 `b`，结果为 `true` |

```
// && 短路
IF : (data != null && len(data) > 0) { ... }
// data 为 null 时，len(data) 不会被求值，避免空指针

// || 短路
IF : (cache == null || cache_expired) { ... }
// cache 不为 null 时，cache_expired 不会被求值
```

> `!`（逻辑非）没有短路概念，它只有一个操作数，求值后取反。

### 3.5 字符串插值（f-string）

以 `f` 开头的字符串字面量支持内嵌表达式：

```
f"<文本> {<表达式>} <文本>"
```

| 段 | 说明 |
|----|------|
| `f` | 前缀标记，紧贴左引号 |
| `<文本>` | 普通字符串文本 |
| `{<表达式>}` | 插值表达式，运行时求值并转为字符串嵌入 |

**示例：**

```
name = "Atomix"
version = 1
greeting = f"Hello, {name} v{version}!"     // "Hello, Atomix v1!"

path = f"/data/{env}/config_{timestamp}.json"  // 拼路径
msg = f"processed {count} items in {elapsed}ms"  // 日志
```

**规则：**

| 规则 | 说明 |
|------|------|
| 前缀 `f` 必填 | 无 `f` 前缀的 `""` 是普通字符串，`{}` 不转义 |
| 可嵌套表达式 | `{...}` 内可以是变量、函数调用、运算等合法表达式 |
| 不可嵌套 f-string | `{...}` 内不可再出现 `f"..."` 字面量 |
| 转义 | 花括号自身用 `\{` `\}` 转义 |
| 类型推断 | f-string 表达式的类型始终为 `string` |

```
// 合法的插值
f"count = {count}"
f"result = {to_string(result)}"
f"path = {base}/{f"sub"} "      // ❌ 不可嵌套 f-string

// 花括号转义
f"curly brace: \{hello\}"        // "curly brace: {hello}"
```

### 3.6 匿名函数（fn 表达式）

#### 模板

```
fn (<参数>) <表达式>
fn (<参数>) { <语句体> }
```

| 段 | 必填 | 说明 |
|----|------|------|
| `fn` | ✅ | 关键字，与普通函数定义的 `fn` 同一语义 |
| `(<参数>)` | ❌ | 参数列表，每个参数 `<类型> <变量名>`，类型可省略（由上下文推导） |
| `<表达式>` | ✅ | 单表达式（二选一），表达式的值即为返回值 |
| `{ <语句体> }` | ✅ | 语句块（二选一），需 `return` 返回值 |

#### 禁止规则

| 写法 | 原因 |
|------|------|
| `fn` 无参数无体 | 无意义 |

> `fn` 表达式与普通函数定义的 `fn` 是同一个关键字，区别仅在于是否有名称。因此不引入"lambda"独立概念，也不使用 `=>` `->` `|x|` 等额外符号。

#### 示例

```
// 做装饰器
HTTP : "url" [fn (data) decrypt(data)] => RAW

// 做 CALL 参数
CALL : transform(data, fn (x) x * 2) => result

// 显式类型标注
fn (int x, int y) x + y

// 带块体
fn (list[int] items) {
    int total = 0
    for (item in items) { total = total + item }
    return total
}
```

## 4. 箭头运算符

### 4.1 语义

箭头是**所有权移动**操作，不是复制。数据从源移动到目标后，源不再持有。

与 `=` 赋值的区别：

| 操作 | 语义 | 源状态 |
|------|------|--------|
| `=` | 复制（copy） | 源仍持有 |
| `=>` / `<=` | 移动（move） | 源不再持有 |

### 4.2 书写方向

`=>` 和 `<=` 是同一个移动操作的两种书写形式，语义完全等价。

```
A => B   ← 数据从 A 移动到 B（右箭头，A 是源，B 是目标）
B <= A   ← 数据从 A 移动到 B（左箭头，A 是源，B 是目标）
```

箭头决定了数据流向，左右元素的顺序与箭头方向联动：

- `=>`：左边是源，右边是目标
- `<=`：右边是源，左边是目标

### 4.3 使用区

| 区 | 可用箭头 | 说明 |
|----|---------|------|
| INPUT | `=>` / `<=` | 数据源移动到变量 |
| TASK | `=>` 仅 | 函数产出移动到变量。`<=` 在此区无语义 |
| WORKS | `=>` / `<=` | 方法内数据流动 |
| TOOLS | `=>` / `<=` | 工具函数内数据流动 |
| OUT | `=>` / `<=` | 变量移动到交付目标 |

### 4.4 示例

```
// INPUT：两种写法等价
HTTP : "https://api.com/data" => RAW : bytes
RAW : bytes <= HTTP : "https://api.com/data"

// TASK：统一用 =>
CALL : fetch_data(INPUT : URL) => raw
CALL : parse(raw) => user_obj

// OUT：两种写法等价
RAW => HTTP : "https://api.com/upload"
HTTP : "https://api.com/upload" <= RAW
```

## 5. WAIT — 任务派发与等待

跨区编排关键字，在 TASK / WORKS / TOOLS 区均可使用。

### 5.1 语法模板

```
WAIT <句柄> = <文件>(<参数>) = <结果变量> IF : (<条件>) { <处理体> }
WAIT <句柄> = <文件>(<参数>) => <结果变量> IF : (<条件>) { <处理体> }
```

### 5.2 段说明

| 段 | 必填 | 说明 |
|----|------|------|
| `WAIT` | ✅ | 关键字，阻塞等待任务完成 |
| `<句柄>` | ✅ | 句柄变量名，标识本次派发的任务实例 |
| `=`（句柄赋值） | ✅ | 固定写法 |
| `<文件>` | ✅ | 目标文件名（`.atx`），需已 USE 或在可发现路径 |
| `(<参数>)` | ❌ | 传入参数列表 |
| `=` / `=>` | ✅ | 结果语义：`=` 复制，`=>` 移动。两者互斥 |
| `<结果变量>` | ✅ | 接收返回结果的变量 |
| `IF : (<条件>)` | ❌ | 状态检查条件，仅限 `IS*` 系列状态字（详见《TASK语法.md》第 3 节） |
| `{ <处理体> }` | ❌ | 条件满足时的处理体 |

### 5.3 参数与 INPUT 的关系

```
WAIT h = compute() => result
// 不传参 → 目标文件 compute.atx 的 INPUT 区正常执行

WAIT h = compute("https://my-api.com", "./cfg.json") => result
// 传参 → 跳过目标文件的 INPUT 区
// 参数按位置映射为常量，注入目标文件的 TASK 区
// 目标文件 INPUT 区的声明相当于"参数默认值申明"
```

### 5.4 返回结果

目标文件的 OUT 区可能有多个产出变量。返回统一为**字典（dict）**：

```
{
    "变量名_1": 值_1,
    "变量名_2": 值_2,
    ...
}
```

| 产出类型 | 返回行为 |
|----------|----------|
| 普通数据（string/number/bool/list/dict） | 值本身 |
| 文件类数据 | 返回文件句柄 |
| 二进制数据（bytes） | 字节序列 |

### 5.5 结果语义：`=` 与 `=>`

| 符号 | 语义 | 源状态 |
|------|------|--------|
| `=` | 复制（copy） | 源仍持有，可多次消费 |
| `=>` | 移动（move） | 源不再持有，所有权转移 |

与第 4 节箭头运算符的语义一致，两者互斥。

### 5.6 阻塞与并发

**WAIT 是阻塞的。** 执行到 WAIT 时，当前执行流停滞，直到目标文件的任务全部执行完毕。

**但 WAIT 并不排斥并发。** 编译器在编译期分析依赖图：

- 多个 WAIT 之间无数据交叉依赖 → 标记为可并行派发
- 调度器按批次执行，同一批次内的 WAIT 同时推进

```
// 用户写的是顺序的：
WAIT h1 = compute_a() => r1
WAIT h2 = compute_b() => r2
CALL : merge(r1, r2) => result

// 编译器分析依赖图：h1、h2 无依赖
// → 调度层并行执行 h1 和 h2
// → 两者都完成后才执行 merge
```

### 5.7 额度约束

WAIT 派发受动态并发引擎约束：

| 场景 | 行为 |
|------|------|
| 额度富余 | WAIT 即刻执行 |
| 额度跑满 | WAIT 排队等待，每释放一个额度则启动一个 |
| 本层全部 WAIT 完成 | 向上层推进（依赖图的上一层） |

额度耗尽时，子任务连第一条指令都不会开始跑。WAIT 的"等待"既包含等待子任务完成，也包含等待额度分配。

### 5.8 可用区

| 区 | 可用？ | 说明 |
|----|--------|------|
| INPUT | ❌ | 数据来源声明，不可编排任务 |
| TASK | ✅ | 主编排区 |
| WORKS | ❌ | 纯计算，不可派发任务 |
| TOOLS | ❌ | 工具函数，不可派发任务 |
| OUT | ❌ | 数据交付，不可编排任务 |

## 6. USE — 文件头引入

文件级声明，出现在所有区之前。不属于任何区。

### 6.1 语法模板

```
USE <FILENAME>
USE <FILENAME> :: <WORKS>
USE <FILENAME> :: <TOOLS>
USE <FILENAME> :: <TOOLS>::<内部工具>
FROM <URL> > USE <FILENAME>
FROM <URL> > USE <FILENAME> :: <WORKS>
FROM <URL> > USE <FILENAME> :: <TOOLS>
FROM <URL> > USE <FILENAME> :: <TOOLS>::<内部工具>
```

### 6.2 段说明

| 段 | 必填 | 说明 |
|----|------|------|
| `FROM <URL>` | ❌ | 远程地址。存在时编译器在编译期拉取文件，拉不到则编译失败 |
| `> USE` | ✅ | `>` 为拉取方向 |
| `<FILENAME>` | ✅ | 文件名（`xxx.atx`） |
| `:: <WORKS>` | ❌ | 精准选取该文件的 WORKS 区 |
| `:: <TOOLS>` | ❌ | 精准选取该文件的 TOOLS 区 |
| `:: <TOOLS>::<内部>` | ❌ | 精准选取 TOOLS 区中的具体工具 |

### 6.3 作用域

- USE 声明在**文件头**，不属于 INPUT / TASK / OUT / WORKS / TOOLS 中的任何一个区
- 引入的内容在当前文件的**所有区**中可见
- 全量引入通过 `文件名.xxx` 前缀引用；精准引入则直接注入命名空间

### 6.4 示例

```
// 文件头：引入所有依赖
USE helper.atx
USE compute.atx :: WORKS
USE util.atx :: TOOLS :: parse_csv
FROM "https://registry.atomix.dev/std" > USE fetcher.atx :: WORKS

// 各区可以直接使用引入的内容
TASK :
    CALL : parse_csv(INPUT : RAW) => data
    WAIT h = compute() => result
```

## 7. RAISE — 抛出异常

抛出异常。可用于 WORKS 和 TOOLS 区（INPUT / TASK / OUT 不可用）。

### 7.1 语法

```
RAISE <异常>
RAISE <异常>("<消息>")
```

| 段 | 必填 | 说明 |
|----|------|------|
| `<异常>` | ✅ | 异常类型，必须是 `EXCEPTION` 定义的异常名 |
| `"<消息>"` | ❌ | 异常描述信息 |

### 7.2 示例

```
TOOLS :
    EXCEPTION ValidationError

    void : fn check_input(dict data) {
        if (!data has "name") {
            RAISE ValidationError("missing name field")
        }
    }
```

```
WORKS : Processor {
    string input

    void : fn process() {
        if (this.input == "") {
            RAISE ValueError("empty input")
        }
    }
}
```

## 10. 文件结构总览

一个 Atomix 文件由以下逻辑区构成。编译器编译时自动按此顺序重排，不限制书写顺序。

```
┌─────────────────────────────┐
│  文件头 USE 声明             │  引入外部依赖（不属于任何区）
├─────────────────────────────┤
│  TOOLS 区                   │  工具函数、泛型、异常定义
├─────────────────────────────┤
│  INPUT 区                   │  数据来源声明，产出文件级常量（不可变）
├─────────────────────────────┤
│  TASK 区                    │  编排逻辑，引用 INPUT 常量，可 WAIT 派发 WORKS
├─────────────────────────────┤
│  WORKS 区                   │  可复用任务模板，可定义多个
├─────────────────────────────┤
│  OUT 区                     │  数据交付，引用 TASK 声明的产出变量
└─────────────────────────────┘
```

**各区约束：**

| 区 | 数量 | 说明 |
|----|------|------|
| `USE`（文件头） | 任意 | 不属于任何区 |
| `TOOLS` | 有且仅有一个 | 工具函数、类型定义 |
| `INPUT` | 有且仅有一个 | 数据来源 |
| `TASK` | 有且仅有一个 | 编排核心 |
| `WORKS` | 任意（0~N） | 可复用任务模板 |
| `OUT` | 有且仅有一个 | 数据交付 |

详细规则见 [`docs/编译行为.md`](../编译行为.md)。

## 11. 补充说明

### 11.1 设计舍离

以下常见语法要素经审视后**明确不纳入**当前设计（理由附后）：

| 要素 | 理由 |
|------|------|
| **枚举类型** | 任务编排中场景窄，状态/分类可由字符串或 int 常量替代 |
| **模式匹配（match）** | IF/ELSE 链已覆盖条件分支，模式匹配引入额外复杂度 |
| **列表推导式** | 在任务编排中无实际场景（非通用计算语言） |
| **三目运算符 `? :`** | IF/ELSE 已足够，三目运算符降低可读性 |
| **try / catch** | CALLIF + IS* 状态字覆盖 TASK 区错误处理；RAISE 覆盖 TOOLS/WORKS |

### 11.2 预留的扩展方向

以下要素当前未设计，但保留在未来补充的可能：

| 要素 | 说明 |
|------|------|
| **break / continue** | FOR 循环缺少退出机制，但需设计 Atomix 特色的形式（非简单照搬） |
| **泛型约束扩展** | 当前 4 种内置约束（Comparable/Numeric/Hashable/Iterable），自定义约束待补充 |
