# Atomix 命令行规范

> 架构版本: v0.2
> 最后更新: 2026-07-16
> 设计哲学: **简单 · 自动 · 智能**
> 配套文档: 参见 外围工具.md（高层概览）

---

## 1. 设计哲学

### 简单

最常见的操作 = 最短的命令。默认行为就是用户最想要的那个。

```
atomix build              ← 编译全部，不需要 --all
atomix runner run         ← 跑全部，不需要 --all
atomix test               ← 跑全部测试
```

### 自动

用户少敲一个字，工具多猜一步。猜错了也有明确的提示和纠偏。

```
atomix run cleanip        ← "没有 cleanup 任务，你是不是想跑 cleanup？"
atomix build              ← 自动找 atomix.toml，自动增量编译
```

### 智能

不把用户的错误当错误——当猜测意图的机会。配置该写哪就写哪，用户不需要操心。

## 2. 语法规则

### 2.1 基本形式

```
atomix <顶级子命令> [--子模式] [-参数] [--子模式] [-参数] ...
```

### 2.2 约定

| 规则 | 说明 | 示例 |
|------|------|------|
| 顶级子命令 | 无前缀，纯单词 | `build`、`runner`、`origin` |
| 子模式切换 | `--` 开头 | `--set`、`--list`、`--status` |
| 子模式参数 | `-` 单字母或 `--` 全词 | `-ip`、`-as`、`--config` |
| 连续模式 | 同一命令可串多个子模式 | `--set -ip 10.0.0.5 -as prod --list` |

### 2.3 参数与值

```
--子模式         → 布尔开关
--子模式 <值>    → 带参数的子模式
-单字母 <值>     → 短参数
--全词 <值>      → 长参数
```

### 2.4 通配规则

绝大多数子命令遵循同一个模式：

```
无参数 = 操作全部
有参数 = 操作指定
```

```
atomix build              → 编译全部
atomix build cleanup      → 只编译 cleanup

atomix runner run         → 跑全部任务
atomix runner run cleanup → 只跑 cleanup

atomix test               → 跑全部测试
atomix test cleanup       → 只跑 cleanup 的测试
```

少数命令（如 `origin`）没有"全部/指定"区分——它们的管理对象是唯一的。

├── origin        远程连接管理

## 4. 子命令详表

### 4.1 `build` — 编译

将 `.atx` 源码编译为 `.atxe` 可执行产物。不出产物时为语法+类型检查。

```
atomix build                     ← 编译全部 Task
atomix build <name>              ← 只编译指定 Task
atomix build <path>              ← 编译指定文件或目录

atomix build --config <path>     ← 指定配置文件
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `<name>` | 位置参数 | Task 名称或文件名（可选） |
| `--config <path>` | 长参数 | 指定 atomix.toml 路径 |

默认行为：自动找项目根目录的 `atomix.toml`，增量编译（只编译有变动的文件）。

---

### 4.2 `runner` — 本地执行引擎

Runner 是执行任务的核心引擎。没有 `--origin` 时所有操作都在本地执行。

```
atomix runner run                    ← 执行全部 Task
atomix runner run <name>             ← 只执行指定 Task
atomix runner run <name> --origin prod  ← 在远程 runner 上执行

atomix runner status                 ← 查看引擎运行状态
atomix runner stop                   ← 停止引擎
atomix runner --config <path>        ← 指定引擎配置
```

| 子模式 | 参数 | 说明 |
|--------|------|------|
| `run` | `[<name>]` | 执行任务。无参=全量 |
| `status` | 无 | 查看引擎状态 |
| `stop` | 无 | 停止引擎 |
| `--config` | `<path>` | 指定配置文件路径 |
| `--origin` | `<remote>` | 指定远程 runner 别名 |

> **run 的特殊性**：`runner run` 是 runner 的子模式（"运行这个引擎"），也是通用的执行入口（"跑任务"）。带 `--origin` 就是在远程跑。

---

### 4.3 `origin` — 远程连接管理

管理远程 runner 的连接别名。**所有配置写本地，不篡改远程。**

```
atomix origin --add -ip <地址> -as <别名> [--port <端口>]
atomix origin --list
atomix origin --status <别名>
atomix origin --log <别名> [--tail <行数>]
atomix origin --remove <别名>
```

| 子模式 | 参数 | 说明 |
|--------|------|------|
| `--add` | `-ip <地址>` `-as <别名>` | 注册远程连接 |
| | 可选 `--port <端口>` | 默认 9000 |
| `--list` | 无 | 列出所有远程别名 |
| `--status` | `<别名>` | 查看远程引擎状态 |
| `--log` | `<别名>` | 拉取远程引擎日志 |
| | 可选 `--tail <行数>` | 只取末尾 N 行 |
| `--remove` | `<别名>` | 删除远程连接 |

配置写入位置：`atomix.toml` 的 `[origin]` 段。

---

### 4.4 `task` — 深度检查 + 脚手架

对任务进行深度检查（内存、数据流、步骤追踪），也可以创建新任务。

```
atomix task add <name>                 ← 脚手架：创建任务模板
atomix task <name>                     ← 本地深度检查
atomix task <name> --origin prod       ← 远程深度检查
atomix task <name> --step <id/name>    ← 检查指定步骤
atomix task <name> --step <id> -mem    ← 看指定步骤的内存
atomix task <name> --step <id> -log    ← 看指定步骤的日志
atomix task <name> --stop --origin prod   ← 停止远程任务
atomix task <name> --restart --origin prod ← 重启远程任务
atomix task <name> --origin prod --config daemon.toml ← 远程改配置（危险，远程可拒绝）
```

| 参数/子模式 | 类型 | 说明 |
|-------------|------|------|
| `add` | 子模式 | 脚手架：创建任务 |
| `<name>` | 位置参数 | 任务名称 |
| `--origin <别名>` | 长参数 | 指定远程 runner |
| `--step <id/名称>` | 长参数 | 聚焦到某一步 |
| `-mem` | 短参数 | 查看内存模型（当前 step 或全局） |
| `-log` | 短参数 | 查看执行日志 |
| `-run` | 短参数 | 重新运行 |
| `-restart` | 短参数 | 重启 |
| `-stop` | 短参数 | 停止 |
| `--config` | 长参数 | 指定配置（远程操作时需权限） |

---

### 4.5 `test` — 测试

测试任务的行为和性能。支持本地测试和远程压测。

```
atomix test                          ← 运行全部测试
atomix test <name>                   ← 运行指定测试
atomix test <name> --origin prod     ← 在远程运行测试
atomix test <name> --origin prod --load  ← 远程压测
atomix test --filter <pattern>       ← 按名称过滤
atomix test --timeout <duration>     ← 单测试超时
```

| 参数/子模式 | 说明 |
|-------------|------|
| `<name>` | 测试名称或文件名 |
| `--origin <别名>` | 指定远程 runner |
| `--load` | 启用压测模式 |
| `--filter <模式>` | 按名称过滤测试 |
| `--timeout <时长>` | 单测试超时时间 |

---

### 4.6 `analysis` — 代码质量

静态检查和代码风格管理。不出产物，只出报告。

```
atomix analysis --lint              ← 代码规范检查
atomix analysis --format            ← 格式化
atomix analysis --type              ← 类型检查（仅分析，不编译）
atomix analysis --safe              ← 安全检查
atomix analysis --complexity        ← 复杂度分析
atomix analysis --fix               ← 自动修复可修复的问题
atomix analysis --check             ← 只检查不修改（CI 模式）
```

| 子模式 | 独立使用 | 组合使用 | 说明 |
|--------|----------|----------|------|
| `--lint` | ✅ | ✅ | 规范检查 |
| `--format` | ✅ | ✅ | 格式化 |
| `--type` | ✅ | ✅ | 类型检查 |
| `--safe` | ✅ | ✅ | 安全检查 |
| `--complexity` | ✅ | ✅ | 复杂度 |
| `--fix` | ❌ | ✅ | 自动修复，不可单独使用 |
| `--check` | ✅ | ✅ | 只读模式 |

组合示例：

```
atomix analysis --lint --type --safe    ← 同时检查规范+类型+安全
atomix analysis --format --fix          ← 格式化并自动修
```

---

### 4.7 `add` — 装包

从远程索引或指定源安装依赖包。

```
atomix add <包名>                     ← 从索引安装最新版本
atomix add <包名>@<版本>              ← 安装指定版本
atomix add <包名> --url <地址>        ← 从指定源安装
atomix add --path <路径>              ← 从本地路径安装
atomix add --dry-run                  ← 预览依赖树，不实际写
```

| 参数 | 说明 |
|------|------|
| `<包名>` | 包名称（可选 @版本） |
| `--url <地址>` | 指定 Git 或 HTTP 来源 |
| `--path <路径>` | 指定本地路径 |
| `--dry-run` | 预览模式，不实际安装 |

> 版本范围遵循 semver 规范：`^1.0`、`~1.2`、`>=1.0 <2.0`。

---

### 4.8 `rm` — 删包

从项目中移除已安装的依赖包。

```
atomix rm <包名>                     ← 移除包
atomix rm <包名> --no-lock           ← 移除包但保留 lock 条目
```

---

### 4.9 `lock` — 锁定依赖

生成或更新 `atomix.lock`，锁定精确的依赖树和完整性哈希。

```
atomix lock                          ← 生成/更新 lock 文件
```

---

### 4.10 `sync` — 同步

同步依赖和远程状态。

```
atomix sync                          ← 同步全部（依赖 + 远程状态）
atomix sync --deps                   ← 仅同步依赖（按 lock 安装）
atomix sync --origin                 ← 仅同步远程 runner 状态
```

---

### 4.11 `cache` — 缓存管理

管理构建产物、临时数据、调试录制轨迹等一切可清理的本地数据。

```
atomix cache                         ← 查看缓存状态（大小、位置）
atomix cache --clean                 ← 手动清理全部缓存
atomix cache --time <时长>           ← 设置自动清理周期
atomix cache --size <大小>           ← 设置缓存上限
atomix cache --dir <路径>            ← 设置缓存目录
```

| 子模式 | 说明 |
|--------|------|
| `--clean` | 手动清理 |
| `--time <时长>` | 自动清理周期（如 `7d`、`30d`） |
| `--size <大小>` | 缓存上限（如 `500MB`、`2GB`） |
| `--dir <路径>` | 缓存目录位置 |

---

### 4.12 `init` — 项目初始化

从零创建一个新项目（项目骨架 + `atomix.toml`）。

```
atomix init                          ← 在当前目录创建项目
atomix init <路径>                   ← 在指定目录创建项目
```

---

### 4.13 `info` — 项目/版本信息

```
atomix info                          ← 当前项目信息
atomix info --version                ← 仅显示版本号
atomix info --verbose                ← 详细诊断信息
```

## 5. 配置解析规则

### 5.1 配置链（优先级从高到低）

| 优先级 | 来源 | 示例 |
|--------|------|------|
| 最高 | 命令行参数 | `--config /path/to/daemon.toml` |
| 中 | 本地配置文件 | `atomix.toml` 中的 `[settings.runtime]` |
| 最低 | 默认值 | 各参数的硬编码默认值 |

### 5.2 本地 vs 远程

```
--origin 存在 → 消息发到远程 runner → 远程使用自己的配置
--origin 不存在 → 所有操作在本地执行 → 使用本地配置
```

### 5.3 远程安全模型

远程 runner 通过配置文件声明禁止的命令列表：

```toml
# /etc/atomix/runner.toml
[security]
deny_commands = ["task --config", "runner stop"]
```

- 本地 CLI 不拦截——用户想敲什么敲什么
- 命令发到远程后，远程 runner 裁决是否允许
- 被拒绝时返回 `403` 错误，本地 CLI 显示拒绝原因

> 安全声明：工具提供完整的能力，不对用户能做或不能做什么做假设。生产环境的安全由运维规则和流程保障。

## 6. 退出码

| 退出码 | 含义 |
|--------|------|
| `0` | 成功 |
| `1` | 一般错误（参数错误、配置缺失等） |
| `2` | 编译错误 |
| `3` | 运行时错误（任务失败、OOM 等） |
| `4` | 远程连接失败 |
| `5` | 远程拒绝（被 deny_commands 拦截） |
| `6` | 超时 |
| `7` | 内部错误（bug） |

| `atomix rm` | 保留 ✓ |
| `atomix lock` | 保留 ✓ |
| `atomix sync` | 保留 ✓ |
| `atomix cache` | 保留 ✓ |
| `atomix init` | 保留 ✓ |
| `atomix info` | 保留 ✓ |

| `atomix-pm` | 归入 `add` / `rm` / `lock` |
| `atomix-format` | 归入 `analysis` |
| `atomix-lint` | 归入 `analysis` |
