# Atomix 运行时架构

> 架构版本: v0.1 (占位)
> 状态: **待专项设计**

---

## 1. 范围

运行时引擎的核心机制设计：

- **并发算法**：动态并发公式、额度计算、任务调度策略
- **任务池**：任务队列、优先级、生命周期管理
- **调度器**：最深层优先调度、批次管理
- **持久运行与热注入**：Daemon 模式下持续接收新任务、任务热加载

## 2. 参考基础

总纲（01-总纲与哲学.md §4）描述了动态并发引擎的基本原理：

- 输入：任务总量 + 宿主机硬件特征
- 计算：动态并发公式 → 最优并发数 N
- 执行：按批次执行，每批 N 个并发

指令集（02-指令集规范.md §3.6）定义了任务管理指令：

- `TASK_FORK` / `TASK_JOIN` / `TASK_RET` / `TASK_SELF`
- IR 的 `.task` 段存放任务定义表与依赖图
