# Atomix 外围工具

> 架构版本: v0.1 (设计阶段)
> 最后更新: 2026-07-16

---

## 1. 二进制分布

| 二进制 | 用途 | 在开发机 | 在生产机 |
|--------|------|----------|----------|
| `atomix` | 主命令——项目管理、编译、构建、依赖、部署 | ✅ | ❌ |
| `atomix run` | 任务执行（开发/测试/CI） | ✅ | 可选 |
| `atomix debug` | 调试器 | ✅ | ❌ |
| `atomix-daemon` | 生产环境持久执行引擎 | 可选 | ✅ |

生产机最小部署：**`atomix-daemon` 二进制 + `.atxe` 产物**。不需要编译器、源码或包管理器。

## 2. `atomix` 主命令

### 2.1 项目管理

```bash
atomix init [path]
    # 在当前目录（或指定路径）创建项目骨架：
    # atomix.toml + .atomix/ + 示例 tasks/ tools/

atomix task add <路径>
    # 自动声明 Task。参数为目录则扫描所有 .atx，为文件则单个。
    # 写入 atomix.toml 的 [[tasks]] 段。
    # 子参数：
    #   --schedule "0 3 * * *"   定时调度
    #   --group "daily"          分组标签
    #   --exclude "*_test.atx"   排除匹配文件

atomix task rm <名称>
    # 从 atomix.toml 中移除某个 Task 声明（不删除源文件）

atomix task list
    # 列出当前项目所有声明的 Task
```

### 2.2 依赖管理

```bash
atomix install index
    # 首次：clone 远程索引仓库 → .atomix/cache/index/
    # 子参数：
    #   --update                git pull 增量更新
    #   --source <url>          指定索引仓库（默认 github:atomix-eco/index）

atomix install [<名称>[@<版本>]]
    # 安装依赖。不带参数则读 atomix.toml 全部安装。
    # 子参数：
    #   --git <url>             指定 Git 来源
    #   --path <path>           指定本地路径
    #   --dry-run               预览依赖树，不实际安装

atomix index --add <路径> --as <别名>
    # 注册本地路径别名（写入当前项目 atomix.toml [aliases]）
    # 严格项目级——不跨项目

atomix index --list
    # 列出当前项目所有别名

atomix index --remove <别名>
    # 移除别名

atomix lock
    # 生成/更新 atomix.lock。锁定每个 Task 的精确依赖树 + 完整性哈希
```

### 2.3 编译与检查

```bash
atomix check
    # 语法检查 + 类型检查，不产出产物

atomix build [--task <名称>]
    # 编译。不指定 --task 则编译全部声明的 Task。
    # 编译阶段：.atx → .atxo；链接阶段：N × .atxo → .atxe

atomix clean
    # 清理 .atomix/build/ 和 output/

atomix lint
    # 代码规范检查

atomix format [--check]
    # 格式化源码。--check 只检查不修改
```

### 2.4 部署与发布

```bash
atomix deploy <Task名> [--target <host>]
    # 编译 + 推送到生产机 Daemon。不指定 --target 则用 atomix.toml 中 [deploy] 的 targets
    #   --all                   部署全部 Task
    #   --target <host>         指定目标 Daemon 地址（覆盖配置文件）

atomix public [--task <名称>]
    # 发布包。不带 --task 则全量发布（整个项目编译为一个 .atxp 归档）
    #   --task <名称>           只发布指定 Task 的产物
    #   --local                 只产出归档，不推送到远程
    #   --output <dir>          指定产出目录
```

## 3. `atomix run` — 任务执行

```bash
atomix run <Task名或文件路径>
    # 编译 + 执行。支持源文件和已编译的 .atxe 产物。

    # 执行范围：
    #   --all                   执行所有声明的 Task

    # 运行时临时覆盖（不进 atomix.toml，仅本次执行生效）：
    #   --max-memory <值>       内存限制
    #   --cpu <核数>             CPU 限制
    #   --timeout <值>          硬超时
    #   --input-size <值>       模拟大数据输入
    #   --env <键=值>           注入环境变量

    # 示例：
    atomix run data_cleanup --max-memory 64MB --cpu 1
    atomix run --all --timeout 60m
```

## 4. `atomix test` — 测试

```bash
atomix test [--task <名称>]
    # 运行测试。不指定 --task 则运行所有测试。

    #   --filter <模式>         按名称过滤测试
    #   --timeout <值>          单测试超时
```

## 5. `atomix debug` — 调试器

> 调试子系统设计复杂，独立二进制。详细规格待专项设计。

```bash
atomix debug <Task名>
    # 启动调试会话。预期功能：
    #   - 断点设置与条件断点
    #   - 单步执行（逐指令 / 逐 Step）
    #   - 寄存器与变量查看
    #   - 调用栈回溯
    #   - 远程调试协议（attach 到生产机 Daemon 中的 Task）
```

## 6. `atomix-daemon` — 生产引擎

```bash
atomix-daemon [--config <path>]
    # 生产环境持久执行引擎。启动后：
    #   1. 读配置文件（默认 /etc/atomix/daemon.toml）
    #   2. 扫描 task_dir 中的 .atxe 产物
    #   3. 按 schedule / trigger 调度执行
    #   4. 监听端口接受 atomix deploy 推送的新任务
```

Daemon 配置（`daemon.toml`——不是 `atomix.toml`）：

```toml
# /etc/atomix/daemon.toml
[daemon]
listen = "0.0.0.0:9000"          # 接受 deploy 推送的端口
task_dir = "/var/atomix/tasks"   # 扫描 .atxe 的目录
state_dir = "/var/atomix/state"  # 运行时状态文件
max_memory = "256MB"
cpu_affinity = [0, 1]
```

## 7. 环境目录

所有构建和运行时产物在 `.atomix/` 下管理（详见 包管理.md §4）：

```
.atomix/
├── cache/         # 易失——垃圾桶，删了可完全重建
│   ├── deps/      # git clone 的依赖
│   ├── index/     # 远程索引缓存
│   └── tmp/       # 编译临时文件
├── build/         # .atxo 中间产物
├── output/        # .atxe 最终产物（部署单元）
└── state/         # 运行时状态
```
